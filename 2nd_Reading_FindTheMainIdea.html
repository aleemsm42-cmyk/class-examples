Teacher Notes & Workflow

How this Workflow Operates:

  • The Prep: You identify the TEK/Standard you are teaching next week.
  • The Customization: You fill in the 6 bracketed variables in the prompt to tell the AI exactly what the theme should be and what 4 stats the game should track.
  • The Generation: Paste it into Gemini, ChatGPT, or Claude. Because the prompt strictly dictates the UI design, the cascading math logic, and the exact copy/paste mechanics you finalized for the Final Report, the AI will build it perfectly on the first try.
  • The Deployment: You copy the raw HTML output, paste it into your Google Site "Embed Code" widget (or Canvas HTML editor), stretch the box so it doesn't scroll, and publish.

AI Generation Prompt Template

Copy and paste the template below into your AI of choice (like Gemini or Claude). Fill in the bracketed info to generate a brand new interactive module instantly:

Act as an Expert Educational Game Developer. I want you to code an interactive, HTML-based branching narrative game (like a "Choose Your Own Adventure" simulator) to teach my students a core academic concept. Here are the specific parameters for this game: Grade Level: [Insert Grade Level, e.g., 8th Grade] Subject: [Insert Subject, e.g., US History] Target Standard (TEKS/NGSS): [Insert Standard, e.g., TEKS 8.4(b) - Causes of the American Revolution] Core Concept: [Insert Core Concept, e.g., The economic and political impact of British taxation] Game Theme/Objective: [Insert Theme, e.g., You are a colonial merchant in Boston trying to keep your business alive while navigating British taxes and rebel boycotts.] 4 Variables to Track: [e.g., Wealth, Loyalist Support, Patriot Support, Smuggling Success] REQUIREMENTS FOR THE HTML/JS/CSS CODE: 1. Code Structure: It must be a single, self-contained HTML file containing inline CSS (for styling) and JavaScript (for logic). I need to be able to copy and paste the raw code directly into an LMS (Canvas/Schoology) or Google Site embed block. Do not use external libraries. 2. UI/Design: Create a clean, modern interface. Include a header with the game title. Below that, include a static "Stats Bar" displaying the 4 variables as progress bars (0-100%). Use a color palette appropriate for the theme. 3. Gameplay Mechanics: * The player will face 5 distinct scenarios (Rounds/Levels). * Give them 3 distinct choices per scenario. * Every choice must mathematically impact the 4 variables. * Include "Cascading Logic": If a foundational variable drops below a critical threshold (e.g., 30%), it should trigger an automatic penalty to the other variables and trigger a text warning on the feedback screen. 4. Feedback Screen: After every choice, hide the scenario screen and show a feedback screen. This screen must explain the real-world educational concept behind what just happened based on their choice. Include a "Continue" button to move to the next round. 5. Final Report & Assessment Export: * After the 5th round, hide the game board and show a Final Report. * Calculate a final score based on their variable averages and assign a letter grade (A-F). * Display a clean text summary of their final stats. * Below the stats, provide a <textarea> prompting the student to write a 1-paragraph reflection based on the Core Concept. * Crucial: Do NOT include any "Restart" buttons or extra UI elements on this final screen. The goal is for the student to easily highlight the final stats and their typed reflection, press Ctrl+C, and paste it into their LMS.

Embedding Instructions

To deploy this learning tool to your students:

  1. Select everything in your code editor and copy the generated raw HTML code.
  2. For Google Sites: Go to the Insert menu, click Embed, select the Embed Code tab, and paste the code.
  3. For Canvas / Schoology: Create a new Page or Assignment. Click the HTML Editor icon (usually `` or a cloud icon), and paste the code.
  4. Drag the edges of the embedded widget so that it's tall enough for students to view the entire game without needing an inner scroll bar.
